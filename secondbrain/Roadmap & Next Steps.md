# 🗺️ Roadmap & Next Steps: แผนการทำงานและสถานะปัจจุบัน

ไฟล์นี้ใช้ติดตามความคืบหน้าของโปรเจคในปัจจุบัน และระบุงานที่จะต้องทำต่อไปในอนาคตเพื่อทำให้โปรเจคนี้สมบูรณ์ยิ่งขึ้น

---

## 🚦 สถานะความคืบหน้าปัจจุบัน (Current Project Status)

*   **แกนหลักและฐานข้อมูล**: 🟩 **เสร็จสมบูรณ์ (90%)**
    *   กำหนด Schema ใน Drizzle ครบถ้วน (ตาราง `organizations`, `tenants`, `rental_units`, `billing_cycles`, `meter_readings`, `invoices`, `invoice_items`, `payments`)
    *   สร้างไฟล์ Migration แรกและกำหนดค่า ORM ไว้เรียบร้อยแล้ว
*   **ระบบ Logic การเงิน**: 🟩 **เสร็จสมบูรณ์ (100%)**
    *   จัดการหน่วยเงินแบบเศษสตางค์ (Satang Basis)
    *   มีฟังก์ชันคำนวณและดึงสถานะอัตโนมัติ (VAT, อัตราส่วนต่างค่าไฟ, รหัสรันนิ่ง)
*   **ระบบหน้าบ้าน (UI Dashboard)**: 🟩 **เสร็จสมบูรณ์ (97%)**
    *   หน้าบ้านแบบรวมทุกอย่างหน้าเดียว (All-in-One Dashboard Workspace) ค้นหา ฟิลเตอร์ข้อมูลได้รวดเร็ว
    *   เชื่อมต่อ Client เข้ากับ Server Actions เพื่อทำ CRUD ได้อย่างไร้รอยต่อ
    *   มีโหมดจำลอง (Demo Mode) ที่พึ่งพิง Local State หากไม่มีฐานข้อมูล
    *   เพิ่มหน้าจอจัดการรอบบิล เลือกรอบบิล และสร้างใบแจ้งหนี้ยกชุดแล้ว
*   **ระบบสิทธิ์ผู้ใช้ (Authentication & Roles)**: 🟨 **ทำแกนหลักแล้ว รอทดสอบกับ Clerk จริง**
    *   ตรวจสิทธิ์ก่อนเข้าหน้าหลักและหน้า print แล้ว
    *   ผู้ใช้คนแรกในฐานข้อมูลถูกตั้งเป็น admin อัตโนมัติ
    *   รองรับ role `admin` และ `staff` จากฐานข้อมูลหรือ Clerk claims
*   **การเชื่อมต่อผู้ให้บริการภายนอก (Third-party integrations)**: 🟨 **พร้อมใช้งาน แต่ยังขาดการกำหนดค่า (Pending Configuration)**
    *   Clerk (ระบบ Auth) และ Cloudinary (ระบบเก็บรูปมิเตอร์) เขียนฟังก์ชันเชื่อมต่อแล้ว รอการระบุ Key ใน `.env.local`

## 📋 แผนงานและสิ่งที่ต้องทำถัดไป (Next Steps Checklist)

เพื่อผลักดันให้แอปพลิเคชันนี้พร้อมสำหรับการใช้งานบน Production (ใช้งานได้จริง) แนะนำให้ทำตามลำดับขั้นตอนดังนี้:

### 1️⃣ ตั้งค่าสภาพแวดล้อมและการ Deploy (Environment & Deployment Setup)
*   [ ] คัดลอกไฟล์ `.env.example` เป็น `.env.local` เพื่อเตรียมค่าตัวแปร
*   [ ] สมัครบริการและกรอก Key สำหรับ:
    *   **Neon Postgres**: นำ Connection string มาใส่ใน `DATABASE_URL` และ `DATABASE_URL_UNPOOLED`
    *   **Clerk**: สมัครโปรเจค นำ Publishable Key & Secret Key มาใส่เพื่อเปิดระบบสมาชิก
    *   **Cloudinary**: สมัครและนำ Cloud Name, API Key, API Secret มาใส่เพื่อเปิดฟีเจอร์อัปโหลดภาพมิเตอร์
*   [ ] ทำการย้าย Schema ลงฐานข้อมูลด้วยการรันคำสั่ง:
    ```bash
    npm run db:generate
    npm run db:migrate
    ```

### 2️⃣ การจัดการประวัติของโปรเจคใน Git (First Git Commit)
*   [x] มี initial commit แล้ว: `a285179 feat: initial commit with billing dashboard, print templates, and obsidian secondbrain`
*   [ ] commit งานล่าสุดเรื่องรอบบิล, ใบแจ้งหนี้ยกชุด, และระบบสิทธิ์ผู้ใช้

### 3️⃣ ฟีเจอร์ที่แนะนำให้พัฒนาเพิ่มเติม (Recommended Feature Backlog)
*   [x] **ระบบเปลี่ยน/เพิ่มรอบบิล (Billing Cycle Management UI)**
    *   เพิ่มปุ่มสร้างรอบบิลใหม่ เลือกรอบบิล และเปลี่ยนสถานะรอบบิลแล้ว
*   [x] **ระบบสร้างใบแจ้งหนี้แบบกลุ่ม (Batch Invoice Generation)**
    *   เพิ่มปุ่มสร้างใบแจ้งหนี้ยกชุดจากพื้นที่เช่าที่มีผู้เช่าอยู่แล้ว
    *   ระบบข้ามรายการที่มีใบแจ้งหนี้ในรอบบิลนั้นอยู่แล้ว
*   [x] **ความปลอดภัยและการกำหนดสิทธิ์ระดับหน้าเว็บ (Page-level Authentication)**
    *   หน้าหลักและหน้า print ตรวจสิทธิ์ผ่าน `requireAppUser()` แล้ว
*   [ ] **ทดสอบครบ flow กับข้อมูลจริง**
    *   รัน migration กับ Neon จริง
    *   ทดสอบ Clerk login และ role จริง
    *   ทดสอบ Cloudinary signed upload จริง
    *   ทดสอบตั้งแต่สร้างรอบบิลจนพิมพ์ใบเสร็จ
*   [ ] **ระบบชำระเงินผ่าน QR Code PromptPay แบบระบุยอด**
    *   *ปัญหาปัจจุบัน*: ผู้ใช้ระบุสถานะชำระเงินเองด้วยการคีย์ข้อมูลในระบบแมนนวล
    *   *แนวทางแก้ไข*: ทำระบบเจนเนอเรต QR Code PromptPay แบบ Dynamic ที่ฝังยอดเงินสุทธิของใบแจ้งหนี้นั้น ๆ เพื่อให้ผู้เช่าสามารถสแกนจ่ายได้ง่ายขึ้น
