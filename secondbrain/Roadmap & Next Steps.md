*   **ระบบ Logic การเงิน**: 🟩 **เสร็จสมบูรณ์ (100%)**
    *   จัดการหน่วยเงินแบบเศษสตางค์ (Satang Basis)
    *   มีฟังก์ชันคำนวณและดึงสถานะอัตโนมัติ (VAT, อัตราส่วนต่างค่าไฟ, รหัสรันนิ่ง)

## 📋 แผนงานและสิ่งที่ต้องทำถัดไป (Next Steps Checklist)

เพื่อผลักดันให้แอปพลิเคชันนี้พร้อมสำหรับการใช้งานบน Production (ใช้งานได้จริง) แนะนำให้ทำตามลำดับขั้นตอนดังนี้:

### 1️⃣ ตั้งค่าสภาพแวดล้อมและการ Deploy (Environment & Deployment Setup)
*   [ ] คัดลอกไฟล์ `.env.example` เป็น `.env.local` เพื่อเตรียมค่าตัวแปร
*   [ ] สมัครบริการและกรอก Key สำหรับ:
    *   **Neon Postgres**: นำ Connection string มาใส่ใน `DATABASE_URL` และ `DATABASE_URL_UNPOOLED`
    *   **Clerk**: สมัครโปรเจค นำ Publishable Key & Secret Key มาใส่เพื่อเปิดระบบสมาชิก
    *   **Cloudinary**: สมัครและนำ Cloud Name, API Key, API Secret มาใส่เพื่อเปิดฟีเจอร์อัปโหลดภาพมิเตอร์
*   [ ] ทำการย้าย Schema ลงฐานข้อมูลด้วยการรันคำสั่ง:
    ```bash
    npm run db:generate
    npm run db:migrate
    ```

### 2️⃣ การจัดการประวัติของโปรเจคใน Git (First Git Commit)
*   [ ] รันคำสั่งคอมมิตครั้งแรกเพื่อจัดเก็บโค้ดตั้งต้นเข้าสู่ Git:
    ```bash
    git add .
    git commit -m "feat: initial project setup with billing, print, and obsidian secondbrain"
    ```

### 3️⃣ ฟีเจอร์ที่แนะนำให้พัฒนาเพิ่มเติม (Recommended Feature Backlog)
*   [ ] **ระบบเปลี่ยน/เพิ่มรอบบิล (Billing Cycle Management UI)**
    *   *ปัญหาปัจจุบัน*: ระบบผูกรอบบิลเริ่มต้นไว้ที่รอบแรกสุด (`data.cycles[0]`) ยังไม่มีหน้าจอให้แอดมินเปิดรอบบิลใหม่ (เช่น ขึ้นเดือนใหม่) หรือปิดรอบบิลเก่า
    *   *แนวทางแก้ไข*: เพิ่มปุ่ม "เปิดรอบบิลใหม่" ในแถบตั้งค่า โดยให้เรียก Server Action ไป INSERT ข้อมูลลงในตาราง `billing_cycles`
*   [ ] **ระบบสร้างใบแจ้งหนี้แบบกลุ่ม (Batch Invoice Generation)**
    *   *ปัญหาปัจจุบัน*: การสร้างใบแจ้งหนี้ต้องกดทำทีละห้อง/ทีละผู้เช่า
    *   *แนวทางแก้ไข*: เพิ่มปุ่ม "ออกใบแจ้งหนี้ทั้งหมดในรอบบิลนี้" โดยระบบจะไปกวาดห้องเช่าที่มีผู้เช่าอยู่ทั้งหมด ดึงค่าเช่าพื้นฐานและค่าไฟที่บันทึกมิเตอร์แล้ว มารวมเป็นใบแจ้งหนี้แบบผสม (Mixed Invoice) ให้แบบอัตโนมัติในคลิกเดียว
*   [ ] **ความปลอดภัยและการกำหนดสิทธิ์ระดับหน้าเว็บ (Page-level Authentication)**
    *   *ปัญหาปัจจุบัน*: หน้าหลัก `/` โหลดข้อมูลทันทีโดยไม่มีการตรวจสอบสิทธิ์การล็อกอิน (ถึงแม้จะห่อ `ClerkProvider` ไว้แล้ว แต่ยังไม่มีการปิดกั้นหน้า)
    *   *แนวทางแก้ไข*: ปรับปรุง `src/app/page.tsx` ให้เรียกฟังก์ชันตรวจสอบเซสชันผู้ใช้จาก Clerk หากไม่ได้ล็อกอินให้รีไดเรกต์ไปที่หน้า Sign In ของ Clerk
*   [ ] **ระบบชำระเงินผ่าน QR Code PromptPay แบบระบุยอด**
    *   *ปัญหาปัจจุบัน*: ผู้ใช้ระบุสถานะชำระเงินเองด้วยการคีย์ข้อมูลในระบบแมนนวล
    *   *แนวทางแก้ไข*: ทำระบบเจนเนอเรต QR Code PromptPay แบบ Dynamic ที่ฝังยอดเงินสุทธิของใบแจ้งหนี้นั้น ๆ เพื่อให้ผู้เช่าสามารถสแกนจ่ายได้ง่ายขึ้น
